### 🌟 협업 전략  

**✔️ 브랜치 구조**   
```bash
├── main
│
├── BE(dev)
│   ├── BE/feat/{이슈번호}
├── FE(dev)
│   └── FE/feat/{이슈번호}
```

<br/>   

**✔️ 이슈 관리** 
```bash
- 이슈에 담겨진 task 단위로 commit
- 라벨 : bug / feature / document
```

<br/>

**✔️ 커밋 컨벤션**   
```bash
[{commit type}] title

내용

[Issue] #{issue number}   
```

<br/>

**✔️ 이슈/마일스톤 생성**   
```bash
[WE]/[FE]/[BE] 제목 
```
